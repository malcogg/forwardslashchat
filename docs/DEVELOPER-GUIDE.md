# Developer Guide — ForwardSlash.Chat

**Audience:** Engineers shipping features, fixing bugs, or onboarding to the codebase.  
**Last updated:** April 2026

For a shorter first-day checklist, still read [DEVELOPER-HANDOFF.md](./DEVELOPER-HANDOFF.md). This guide goes deeper on **how** the app is organized and **how** to work on it safely.

---

## 1. Clone and run

```bash
git clone <repo-url>
cd forwardslashchat
npm install
cp .env.example .env.local   # then fill values — see DEVELOPER-HANDOFF / README
npm run db:push          # sync Drizzle schema to dev DB (Neon recommended)
npm run dev
```

Open `http://localhost:3000`.

**Build before PRs:**

```bash
npm run build
npm run lint
```

## 2. Environment variables (summary)

| Variable | Role |
|----------|------|
| `DATABASE_URL` or `POSTGRES_URL` | Postgres (Neon pooled URL in prod). |
| Clerk `NEXT_PUBLIC_*` + `CLERK_SECRET_KEY` | Auth. |
| `OPENAI_API_KEY` | Chat completions (customer + demo LLM path). |
| `STRIPE_*` | Checkout + webhook verification. |
| `FIRECRAWL_API_KEY` | Site scan / crawl. |
| `RESEND_API_KEY` | Transactional email. |
| `ADMIN_EMAILS` | Comma-separated; gates `/admin` and some APIs. |
| `CRON_SECRET` | Authorize cron routes. |
| `JOBS_STUCK_AFTER_MINUTES` | Default `90`. Cron job worker marks `running` jobs older than this as failed/requeued (crash/timeout recovery). |
| `SKIP_PAYMENT_RECEIVED_EMAIL` | Set `1` or `true` to skip Resend “Payment received” after Stripe checkout (see `lib/send-payment-received-email.ts`). |
| `VERCEL_ACCESS_TOKEN` / `VERCEL_PROJECT_ID` | Optional domain automation. |

**Demo-specific (optional tunables):**

| Variable | Default / notes |
|----------|-----------------|
| `DEMO_CHAT_MAX_TOKENS` | Caps demo LLM output (see `app/api/chat/demo/route.ts`). |
| `DEMO_CHAT_RATE_LIMIT_PER_MINUTE` | IP rate limit for `/api/chat/demo`. |
| `DEMO_LEAD_RATE_LIMIT_PER_MINUTE` | IP rate limit for `POST /api/chat/demo/lead`. |
| `RESOLVE_HOST_RATE_LIMIT_PER_MINUTE` | Default `60`. IP rate limit for `GET /api/chat/resolve-by-host` (middleware forwards visitor IP). |

Full tables: [DEVELOPER-HANDOFF.md](./DEVELOPER-HANDOFF.md), [TECH-SPEC.md](./TECH-SPEC.md).

## 4. Demo page (`/chat/demo`) — behavior for developers

1. **Lead capture (first visit per browser tab session):** Name → email → phone (optional). Persists via `POST /api/chat/demo/lead` into `demo_chat_leads`. Skippable at each step.
2. **Session gate:** `sessionStorage` key `fs_demo_lead_v2` skips the intro on return within the same tab. **`?forceLead=1`** clears storage and shows the intro again (see footer link on main demo).
3. **Chat:** Hybrid **keyword** responses (instant, with CTA pills) + **LLM** via `/api/chat/demo` for open questions (grounded in `data/demo-content.json`).
4. **New chat (+):** Clears `useChat` messages, stops streaming, resets input — does **not** reset lead capture (use “Show intro again” for that).

## 5. Database workflow

- **Drizzle push (dev):** `npm run db:push` — fast iteration; aligns schema from `db/schema.ts`.
- **Production migrations:** Follow [SETUP-DATABASE-MIGRATIONS.md](./SETUP-DATABASE-MIGRATIONS.md). Hand-written SQL lives in `docs/migrations/` (numbered); apply in order in Neon when adding columns not yet generated by Drizzle.

## 6. Git and deployment

- **Default branch:** `main` (confirm with team).
- **Vercel:** Connect repo; set env vars per environment; configure Stripe webhook URL to `/api/webhooks/stripe`.
- **Post-deploy smoke:** Homepage, `/chat/demo` (lead + message), sign-in, `/dashboard` with test user.
